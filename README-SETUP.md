# Настройка проекта — пошагово

Эти файлы — не полный проект, а Java-код и ресурсы, которые нужно
положить поверх официально сгенерированного шаблона. Так безопаснее:
build.gradle/gradle.properties у Fabric обновляются буквально каждую
неделю (версии Loom, Loader, Fabric API), и если я впишу цифры вручную,
велик шанс, что через месяц они устареют. Официальный генератор всегда
отдаёт актуальные версии под выбранную версию игры.

## 1. Требования

- **JDK 25** (именно 25, не 21 и не 17 — 26.2 требует ровно её).
  Скачать: https://adoptium.net/temurin/releases/?version=25
- **IntelliJ IDEA Community Edition**, версия 2025.3 или новее
  (более старая может не собрать проект на Java 25).
  https://www.jetbrains.com/idea/download/

## 2. Генерация базового проекта

1. Открыть https://fabricmc.net/develop/template/
2. Заполнить форму:
   - Mod Name: любое, например `Tunnel Auger`
   - Mod ID: `tunnel_auger` (если поменяешь — поменяй и пакет в шаге 3)
   - Package Name: `com.example.tunnelauger` (тоже можно своё)
   - Minecraft Version: `26.2`
   - Fabric API: оставить включённым — это единственная внешняя
     зависимость, которая нам нужна, всё остальное решает сам тулчейн.
   - Split client/common sources: неважно, что выберешь — у нас пока
     нет клиент-специфичного кода.
3. Скачать zip, распаковать в рабочую папку.

## 3. Наложить файлы из этого архива

Скопировать содержимое `src/main/...` из этого архива в такую же
папку внутри сгенерированного проекта, заменяя/добавляя файлы:

```
src/main/java/com/example/tunnelauger/TunnelAugerMod.java
src/main/java/com/example/tunnelauger/item/ModItems.java
src/main/resources/assets/tunnel_auger/lang/en_us.json
src/main/resources/assets/tunnel_auger/lang/ru_ru.json
```

Если при генерации ты указал другой Mod ID или пакет — поправь пути
и `package com.example.tunnelauger...` / `TunnelAugerMod.MOD_ID` под себя.

## 4. Прописать точку входа

Открыть `src/main/resources/fabric.mod.json` из сгенерированного
проекта и в `"entrypoints"` → `"main"` добавить наш класс:

```json
"entrypoints": {
  "main": [
    "com.example.tunnelauger.TunnelAugerMod"
  ]
}
```

Остальное (id, depends, minecraft version) генератор уже заполнил
правильно — трогать не нужно.

## 5. Открыть и собрать

1. В IntelliJ: File → Open → выбрать папку проекта.
2. Дождаться синхронизации Gradle (первый раз — 5–15 минут, скачивает
   Minecraft и маппинги, это нормально).
3. Справа вкладка Gradle → Tasks → fabric → `runClient`
   (или в терминале `./gradlew runClient`).
4. Откроется тестовый клиент. Зайти в творческий режим, вкладка
   Tools & Utilities — там должен появиться "Tunnel Auger" /
   "Земляной бур" с фиолетово-чёрной текстурой отсутствия текстуры.
   Это ожидаемо: модель/текстуру добавим следующим шагом, сейчас
   важно, что предмет вообще зарегистрировался и появился в игре.

## Если что-то не компилируется

Импорты вроде `net.minecraft.core.registries.BuiltInRegistries` или
`net.minecraft.resources.Identifier` — это те имена классов, которые
использует официальная документация Fabric на момент 26.2. Названия
внутренних классов игры иногда чуть смещаются между патч-версиями.
Если IntelliJ подчёркивает импорт красным:

1. Поставь курсор на неразрешённое имя класса (например `Identifier`)
   и нажми `Alt+Enter` → IntelliJ покажет варианты автоимпорта из
   реально существующих в этой версии игры классов.
2. Если вариантов несколько — открой официальный туториал
   https://docs.fabricmc.net/develop/items/first-item и сверься
   с актуальным именем.

Это единственное действительно хрупкое место во всей конструкции —
дальше (сам блок-камень, компонент уровня, обработчик копки) я буду
проверять по той же документации перед тем как присылать код.
