# Сборка и запуск проекта

Это **полный Gradle-проект** — ничего накладывать поверх шаблона не нужно.
Описание самого мода, механик и структуры кода — в `README.md`.

## 1. Требования

- **JDK 25** (именно 25 — Minecraft 26.2 требует ровно её).
  Скачать: https://adoptium.net/temurin/releases/?version=25
- **IntelliJ IDEA** 2025.3+ (более старая может не собрать проект на Java 25).

Версии зависимостей зафиксированы в `gradle.properties`:
Minecraft 26.2, Loader 0.19.3, Loom 1.17, Fabric API 0.155.2+26.2.
Свежие значения — на https://fabricmc.net/develop

## 2. Открыть и запустить

1. IntelliJ: **File → Open** → папка проекта.
2. Дождаться синхронизации Gradle (первый раз 5–15 минут —
   качаются Minecraft и маппинги, это нормально).
3. Запуск тестового клиента: вкладка Gradle → Tasks → fabric → `runClient`
   (или в терминале `./gradlew runClient`).

⚠ В dev-запуске порог апгрейда снижен до 15 блоков для быстрого
тестирования (`AugerProgress.DEV_BLOCKS_FOR_UPGRADE`). В собранном jar
действуют боевые пороги 300/1500/5000.

## 3. Быстрая проверка в игре

1. Творческий режим → вкладка **Tools & Utilities**: там лежат все
   4 тира Крушителя (можно тестировать без гринда).
2. Философский камень строителя — вкладка **Functional Blocks**.
3. В выживании: бросить на камень железную кирку + 4 железа + алмаз —
   через полсекунды выпадет Крушитель.
4. Рецепты ритуалов можно менять без пересборки: править JSON в
   `src/main/resources/data/tunnel_auger/ritual_recipes/` и выполнить `/reload`
   (в dev-клиенте ресурсы подхватываются из папки проекта).

## 4. Сборка релизного jar

```bash
./gradlew build
```

Готовый мод: `build/libs/tunnel-auger-<версия>.jar` (файл без `-sources`).
Перед релизом пройти чек-лист в конце `README.md`.

## Если что-то не компилируется

Имена внутренних классов игры (маппинги) иногда смещаются между
патч-версиями. Если IntelliJ подчёркивает импорт красным:

1. Курсор на неразрешённое имя → `Alt+Enter` → выбрать автоимпорт
   из реально существующих в этой версии классов.
2. Свериться с документацией: https://docs.fabricmc.net/develop/
